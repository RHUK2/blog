## Nginx 개요

Nginx는 높은 부하와 많은 클라이언트 연결을 안정적으로 처리하는 웹 서버다. 웹 서버, 리버스 프록시, 메일 서버로 동작하도록 설정할 수 있다.

## 블록 구성

Nginx는 요청 처리 설정을 블록 단위로 계층적으로 관리한다.

- 서버 블록(`server`): 가상 서버 단위의 설정 블록이다. 도메인, 포트, IP 주소에 따라 어떤 서버 블록이 요청을 받을지 결정됨.
- 로케이션 블록(`location`): 서버 블록 내부에 위치하며, 특정 URI 패턴에 대한 요청 처리 방법을 정의함.

## 서버 블록 선택 과정

복수의 서버 블록이 있을 때 Nginx는 다음 순서로 처리할 블록을 결정한다.

1. `listen` 지시어 분석: 요청의 IP와 포트를 서버 블록의 `listen` 값과 비교하여 후보 블록을 추림. 가장 구체적으로 매칭된 블록이 우선됨.

2. `server_name` 지시어 평가: 요청의 `Host` 헤더 값을 각 서버 블록의 `server_name`과 비교함.
   - 정확히 일치: 첫 번째 매칭 블록 선택
   - 앞쪽 와일드카드(`*.example.com`): 가장 긴 매칭 선택
   - 뒤쪽 와일드카드(`example.*`): 가장 긴 매칭 선택
   - 정규 표현식: 첫 번째 매칭 선택
   - 아무것도 매칭되지 않으면 해당 IP/포트의 기본 서버 블록이 처리함.

## 로케이션 블록 매칭

서버 블록이 결정된 후, 해당 요청을 처리할 가장 적합한 로케이션 블록을 선택한다.

### 로케이션 블록 구문

```nginx
location <optional_modifier> <uri_pattern> {
    # 처리 지시어
}
```

수정자(modifier) 종류:

| 수정자 | 설명 |
| --- | --- |
| (없음) | 접두사 매칭 — URI 시작 부분과 비교 |
| `=` | 정확히 일치하는 경우 매칭 |
| `~` | 대소문자 구분 정규식 매칭 |
| `~*` | 대소문자 구분 없는 정규식 매칭 |
| `^~` | 가장 긴 접두사 매칭 후 정규식 평가를 건너뜀 |

### 매칭 우선순위

1. `=` 수정자로 정확히 일치하는 블록이 있으면 즉시 선택함.
2. 가장 긴 접두사 매칭 블록을 찾음. `^~` 수정자가 있으면 즉시 선택함.
3. 설정 파일의 순서대로 정규식 블록을 평가하여 첫 번째 매칭된 블록을 선택함.
4. 정규식 매칭이 없으면 가장 긴 접두사 매칭 블록을 사용함.

일부 지시어(`index`, `try_files`, `rewrite`, `error_page`)는 내부 리다이렉트를 발생시켜 로케이션 블록을 재검색할 수 있다.

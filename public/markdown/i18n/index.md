## i18n과 L10n의 차이

소프트웨어의 전 세계 시장 대응을 위한 두 가지 상호 보완적인 과정임.

- 국제화(Internationalization, i18n): 소프트웨어가 다양한 언어와 지역에 적응할 수 있도록 설계 및 개발하는 과정임. 특정 언어에 의존적인 부분을 분리하고 외부화하여 기술적 토대를 마련함.
- 지역화(Localization, L10n): 국제화된 소프트웨어를 특정 지역의 언어, 문화, 관습 등에 맞게 최적화하는 과정임. 텍스트 번역뿐만 아니라 날짜/시간 형식, 화폐 단위, 문화적 맥락 등을 조정함.

## ICU 메시지 포맷(ICU Message Format)

복잡한 문자열 데이터를 동적으로 처리하기 위한 표준 규격임. 단순 치환을 넘어 문법적 규칙(복수형, 성별 등)을 처리함.

- 변수 치환(Variables): `{name}` 형태로 값을 삽입함.
- 복수형 처리(Pluralization): 수량에 따라 다른 메시지를 출력함.
  - `{count, plural, =0 {no items} one {1 item} other {# items}}`
- 선택형 처리(Select): 성별이나 범주에 따라 메시지를 분기함.
  - `{gender, select, male {He} female {She} other {They}}`
- 숫자/날짜 포맷팅: 지역 설정에 맞춰 숫자나 날짜 형식을 자동으로 조정함.

## 사용자 국가 추측 우선순위

사용자의 거주 국가를 판별하기 위한 전략임.

1. 사용자 선택: 설정 페이지 등에서 직접 입력받은 정보 (정확도 가장 높음).
2. Geolocation API: 브라우저의 위경도 정보 (사용자 권한 필요).
3. IP 기반 위치(IP Geolocation): 서버로 전달된 IP 주소를 데이터베이스와 대조함.
4. 시간대(Timezone): 시스템의 시간대 설정을 통해 유추함.
5. 로케일(Locale): 브라우저 설정 언어를 기반으로 함.

## 사용자 언어 추측 우선순위

콘텐츠 제공을 위한 최적의 언어를 선택하는 전략임.

1. 사용자 명시적 선택: 서비스 내 언어 변경 메뉴를 통한 설정.
2. URL 경로 및 파라미터: `/ko/`, `?lang=en` 등 URL에 포함된 정보.
3. Accept-Language 헤더: 브라우저가 선호하는 언어 목록.
4. 로케일 속성: `navigator.languages` 정보를 활용함.
5. 기본값(Fallback): 매칭되는 언어가 없을 때 제공할 기본 언어 (주로 영어).

## ISO 8601

날짜와 시간 표기를 위한 국제 표준 규격임. 데이터 교환의 정합성을 위해 널리 사용됨.

- 형식: `YYYY-MM-DDTHH:mm:ss.sssZ`
  - `T`: 날짜와 시간의 구분자.
  - `Z`: UTC(협정 세계시) 기준임을 나타냄.
- 특징:
  - 사전순 정렬이 시간 순서와 일치함.
  - 타임존 오프셋을 명확히 표기하여 혼동을 방지함.
  - 대다수의 프로그래밍 언어와 데이터베이스에서 표준으로 지원함.
